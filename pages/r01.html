[CSS] 디버깅 시간을 절약 할 수있는 CSS 명명 규칙


다중선택자 보다, BEM방식에 맞게 선택자 사용 및 그에 따라 스타일 선언

CSS에 문서 구조를 담지 말자

html 구조의 시맨틱과, css 구조의 시맨틱은 별도로
.btn-wrap button.left span ❌
.btn-wrap .btn .btn--accept ⭕

지나친 원자화 금지

.mt-10,.mt-20,.mt-30 ❌
.mt--default,.mt--primary,mt--secondary ⭕

재사용및 결합 가능한 형식

타입 선택자[h1, p 같은 태그 선택자]보다는 클래스를 사용
클래스와 태그선택자를 결합 금지 ex) ul.nav
태그선택자를 자손으로 사용 금지 ex) .nav li

javascript용 클래스

ex) js-active, 스타일을 사용하지 않는다.

멀티 클래스 형식

"골격(width,height)과 모양(background,color)을 분리 "
.btn { /* 버튼 템플릿 스타일 */ }
.btn-primary { /* 주요 버튼과 관련된 스타일 */ }

클래스명 구조화 B(lock) E(element) M(odifier)

/* 유틸리티 */
.ui-utilityName {}

/* 컴포넌트 */
.ComponentName {}

carousel
page-nav
form


/* 컴포넌트 수식어 */
.ComponentName--modifierName {}

form--simple
btn--default{...}
btn--secondary{...}

<div class="btn btn--secondary"></div>

/* 컴포넌트 자손 */
.ComponentName__descendant {}

form__input
form__select
carousel__inner
carousel__item


/* 컴포넌트 자손 수식어 */
.ComponentName__descendant--modifierName {}

form__input--readonly
form__select--disabled
carousel__item--next
carousel__item--prev


/* 컴포넌트 상태 (컴포넌트 범위에서) */
.ComponentName.is-stateOfComponent {}
